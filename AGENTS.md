## Agent skills

### Issue tracker

Issues and PRDs live as local markdown files under `.scratch/<feature-slug>/`. See `docs/agents/issue-tracker.md`.

For any large product, architecture, or interaction change, update the relevant PRD or issue first before changing implementation code. If the change alters product semantics, acceptance criteria, or user-facing behavior, the documentation update must precede the code change and be committed separately when practical.

### Triage labels

This repo uses the default mattpocock/skills triage label vocabulary. See `docs/agents/triage-labels.md`.

### Domain docs

This repo uses a single-context domain documentation layout. See `docs/agents/domain.md`.
